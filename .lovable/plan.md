

## 🔍 نقض التقييم من وجهة نظر المستخدم

### ما يراه المستخدم العادي عند زيارة /pricing الآن (بعد التغييرات السابقة)
1. عدّاد مشتركين + شريط مقاعد متبقية + رسالة "سعر مجمّد" → **إثبات اجتماعي قوي ✅**
2. ثلاث بطاقات: مجاني / احترافي (شارة "الأكثر اختياراً" + حد primary) / أعمال (بدون أي تمييز)

### نقاط الضعف الحقيقية من زاوية المستخدم
| المشكلة | الأثر النفسي |
|---|---|
| Pro مميّز بحدّ خفيف فقط، بدون رفع بصري واضح | لا يبدو "البطل" — يبدو مجرد "خيار وسط" |
| Business بلا أي تمييز | يبدو "للأثرياء فقط" أو "غير ضروري" → المستخدم يتجاهله |
| قائمة المزايا مسطحة وطويلة (8 بنود) | يصعب المسح السريع — المستخدم يقرأ أول 3 ويتجاوز |
| لا يوجد "Value Anchor" (ر.س/يوم أو ر.س/متجر) | السعر يبدو مجرد رقم بدل قيمة |
| لا توجد مقارنة Pro→Business | المستخدم لا يفهم "ماذا أربح بالترقية" |
| نفس CTA لكل البطاقات تقريباً | لا توجد تجربة تدرّجية |

### القرار
تنفيذ خطة التمييز التدرّجي **(Tiered Visual Hierarchy)** التي اقترحناها:
- **Pro = البطل** (scale 105، glow، gradient، value line يومي، social proof)
- **Business = الفاخر التخصصي** (scale 102، gold border، value line لكل متجر، bonus حصري)
- **Free = هادئ** (بدون scale، رمادي خفيف)
- مزايا مجمّعة بفئات بأيقونات (بدل قائمة مسطحة)
- شريط مقارنة Pro→Business
- قسم "لماذا Pro؟ / لماذا أعمال؟"
