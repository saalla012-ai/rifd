# 🔍 مراجعة نقدية كاملة لموقع رِفد

راجعت كامل قاعدة الكود، قواعد البيانات، RLS، routes، server functions، و الـUX. هذي الفجوات والأخطاء مرتبة حسب الأولوية.

---

## 🔴 P0 — أخطاء حرجة تكسر الميزات الأساسية

### 1. توليد النص/الصورة معطّل تماماً (السبب الذي تواجهه الآن)
**الملف:** `src/server/ai-functions.ts`
- يستخدم `supabaseAdmin` من `client.server.ts` الذي يتطلب `SUPABASE_SERVICE_ROLE_KEY` كـ `process.env`
- بيئة TanStack Start Worker **لا تحقن** `SUPABASE_SERVICE_ROLE_KEY` تلقائياً → كل طلب توليد يرمي `Missing Supabase server environment variables`
- **الحل:** استبدال `supabaseAdmin` بـ `context.supabase` (المصادَق من middleware)، RLS كافية لجميع العمليات.

### 2. `usage_logs` بدون policies للكتابة
**الملف:** `supabase/migrations/...` (الجدول الأصلي)
- RLS مفعّل فقط على SELECT، لا توجد INSERT/UPDATE → حتى لو أصلحنا (1)، لن يستطيع المستخدم زيادة عدّاد الاستخدام
- **الحل:** إضافة policies `INSERT`/`UPDATE` على `usage_logs` للمالك (`auth.uid() = user_id`)

### 3. `dashboard.library.tsx` يجلب توليدات كل المستخدمين (بدون فلتر user_id)
- السطر 40-44: `supabase.from("generations").select("*").order(...).limit(100)` بدون `.eq("user_id", user.id)`
- يعتمد على RLS فقط — يعمل لكن من ناحية النية البرمجية مفقود؛ ولو أُضيف بشكل صريح يصبح أسرع وأقل خطأ.
- نفس المشكلة في `dashboard.index.tsx` (favs و recent بدون فلترة صريحة)

---

## 🟠 P1 — أخطاء UX/سلامة بيانات

### 4. `dashboard.settings.tsx` كله **mock data**
- يعرض "أحمد العتيبي" / "ahmed@example.com" بقيم defaultValue ثابتة
- زر الحفظ لا يفعل شيء، Switch بدون state
- صفحة "إعدادات" واجهتك الأولى للحساب → انطباع مكسور

### 5. تناقض حدود الباقة المجانية
- في `pricing.tsx`: "5 توليدات نص + **2 صور** مجاناً"
- في `auth.tsx`: "5 توليدات نص + **2 صورة**"
- في `dashboard.index.tsx` و `ai-functions.ts`: `free: { text: 5, image: **0** }` ← أصفار!
- المستخدم يسجل متوقع 2 صور، يجد 0 → `QuotaExceededDialog` فوراً → خداع

### 6. الـ Onboarding يعرض demo نص محلي (مش AI حقيقي)
- `onboarding.tsx` step 5 يستخدم `generateDemoResult()` من `lib/demo-results.ts`
- التعليق نفسه يقول: "سيستبدل بـAI حقيقي في المرحلة C" — لم يستبدل
- المستخدم الجديد يظن أن AI ضعيف لأن النتيجة قالب محلي

### 7. `dashboard.tsx` (الـ layout) فقط `<Outlet/>` بدون gating
- `DashboardShell` نفسه يحوي gating، لكن الـ layout route مكان أنسب → تكرار logic في كل صفحة

### 8. حقل `created_at` مفقود من type `Profile` بـ `use-auth.tsx`
- الجدول يحتوي `created_at` لكن type في الفرونت ينقصه — شكلي لكن يربك TS

## 🟡 P2 — فجوات استراتيجية / منتجية

### 9. لا يوجد **password reset page**
- `forgot-password.tsx` موجود لكن `reset-password.tsx` موجود — تأكد أنه يتعامل مع `type=recovery` بشكل صحيح (لم أتحقق بعد)

### 10. لا توجد **email verification flow**
- المستخدم يسجل بـ password ويُحوَّل لـ /onboarding مباشرة بدون انتظار التحقق — لو إيميل معطّل لن يصل تأكيد

### 11. تكرار صفحات: `/library` (marketing) و `/dashboard/library` (user)
- صحيح وظيفياً، لكن `/dashboard/templates` و `/library` متشابهتان في الغرض → ممكن تبسيط

### 12. **vs-chatgpt** و **about** و **legal** لم أتحقق منها بعمق

### 13. لا يوجد monitoring/analytics للأخطاء
- لا Sentry، لا logging حقيقي للأخطاء في الـ frontend

### 14. **Dark mode toggle** موجود في الهيدر لكن `use-theme.ts` لم أراجعه — قد يكون بدون persist

### 15. الميزة المُعلَن عنها "تعديل صور المنتجات" لا توجد UI لها أبداً
- مذكورة في `pricing.tsx` (سطر 63: "تحسين صور المنتجات")
- لا route، لا component، لا server function لها

---

## 🟢 P3 — تحسينات جودة

### 16. `dashboard-shell.tsx` يستعلم role في كل mount
- لا cache، لا context → استعلام DB في كل صفحة dashboard

### 17. نص "1 من الشهر القادم" في `dashboard.usage.tsx` خطأ نحوي
- العدّاد UTC بينما المستخدم سعودي (GMT+3) → خلاف 3 ساعات في رأس الشهر

### 18. لا توجد realtime updates لـ usage_logs
- بعد توليد، الـDashboard لا يُحدَّث إلا بـ refresh

### 19. صورة OG مفقودة في كل routes (المعرفي يقول لا تضع شامل، ضع لكل route)
- لا og:image على أي صفحة → روابط واتساب بدون ثَمبنيل

### 20. `text_count` للـ free: حد 5 لكن الـ marketing copy في عدة مواضع يقول "غير محدود" للـ pro — متسق لكن "1000 شهرياً" في الكود ليس "غير محدود"
