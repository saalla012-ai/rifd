
# خطة: حزمة مراجعة "رفد" لـ Claude Max

## السياق
- اشتراك Claude Max لا يدعم API، لكنه يقرأ روابط GitHub العامة + الملفات المرفوعة في المحادثة.
- الريبو مربوط وعام → نستفيد من قراءة Claude المباشرة للكود.
- أولوية المراجعة: **تجربة المستخدم ومنطق الأعمال** (Onboarding, الحصص, الفوترة, تدفق الحملات, لغة المنتج).

## المخرجات

### 1. `rifd-business-context.pdf` (~30-40 صفحة)
وثيقة مركّزة تشرح ما لا يستطيع Claude استنتاجه من الكود وحده:

**القسم أ — هوية المنتج**
- ما هي رفد؟ من العميل المستهدف (تجار التجزئة السعوديون)؟
- لغة المنتج الموحدة (نبرة "اكتب نصاً يبيع، صمّم صورة إعلان") + قيود اللغة.
- المنافسون والتمايز.

**القسم ب — رحلة المستخدم الكاملة**
- التسجيل → التحقق → Onboarding → ملء بيانات المتجر → أول توليد → الترقية.
- نقاط الاحتكاك المعروفة.
- معدلات التحويل المستهدفة لكل خطوة.

**القسم ج — منطق الحصص والفوترة**
- لماذا الحصص اليومية بدلاً من النقاط للنصوص/الصور؟
- نظام نقاط الفيديو (لماذا منفصل؟).
- الباقات (Free, Starter, Growth, Pro) — السقوف والأسعار.
- تدفق OCR للحوالات البنكية + الموافقة اليدوية.
- ضريبة القيمة المضافة السعودية + الفواتير.

**القسم د — ميزات ذكاء الأعمال**
- Campaign Studio (الفلسفة + التدفق).
- Store Memory (كيف تتعلم الـ AI من المتجر).
- Smart Context للحملات.
- Templates Library.

**القسم هـ — الأسئلة المفتوحة (لـ Claude)**
- 15-20 سؤالاً موجّهاً يطلب من Claude الإجابة عليها (مثلاً: "هل تدفق Onboarding يصل بنسبة جيدة للتوليد الأول؟ ما الاحتكاكات؟").

### 2. `rifd-database-schema.sql`
- تصدير كامل من Supabase: جميع الجداول، RLS policies، functions، triggers، enums.
- تعليقات عربية على الجداول الأساسية تشرح دورها.

### 3. `rifd-claude-prompt.md`
برومبت جاهز للصق مباشرة في Claude Max يحتوي على:
- رابط GitHub.
- تعليمات قراءة الـ PDF و SQL أولاً.
- أوامر صريحة بالتركيز على UX + Business Logic.
- إطار عمل المراجعة (نقاط القوة، نقاط الضعف، توصيات مرتبة بالأولوية).
- صيغة الإخراج المطلوبة (تقرير منظم، قابل للتنفيذ).

## آلية التنفيذ التقني

1. **استخراج Schema**: استعلام `pg_dump`-style عبر `supabase--read_query` للحصول على بنية الجداول والـ policies.
2. **بناء PDF**: مكتبة `docx` ثم تحويل عبر `libreoffice --headless` (نفس المسار الناجح في `rifd-technical-brief.pdf` السابق) لضمان عرض عربي صحيح بخط Amiri و RTL.
3. **QA بصري**: تحويل كل صفحات PDF إلى صور و فحصها قبل التسليم.
4. **حفظ في** `/mnt/documents/`.

## التسليم النهائي
3 ملفات + شرح موجز بكيفية استخدامها مع Claude Max خطوة بخطوة (رفع الملفات، لصق البرومبت، انتظار التحليل).
