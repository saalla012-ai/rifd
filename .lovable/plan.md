أفهم رغبتك في حفظ كل شيء حتى لا يضيع، لكن حتى لو قلت “على مسؤوليتي” لا يمكنني وضع كلمات المرور أو مفاتيح API السرية أو مفاتيح الخدمة داخل ZIP أو داخل ملفات المشروع. هذا ليس إجراءً شكلياً؛ هذه القيم تمنح وصولاً مباشراً للحسابات، البريد، قاعدة البيانات، خدمات الذكاء الاصطناعي، Telegram، والفوترة/التخزين. إذا رُفع الملف بالخطأ أو أُرسل لشخص آخر، قد يتم اختراق المشروع بالكامل.

سأجهز لك بدلاً من ذلك حزمة حفظ كاملة وآمنة تحفظ كل ما يمكن حفظه من المشروع، وتترك الأسرار موثقة بالأسماء فقط حتى تعيد إدخالها من مكان آمن عند الحاجة.

## ما سيتم إنشاؤه بعد الموافقة

1. **ZIP كامل للكود والمشروع**
   - يشمل كل ملفات المشروع القابلة للحفظ: `src`, `supabase`, `public`, `remotion`, `tests`, إعدادات البناء، التوثيق، صفحات الموقع، المكونات، الدوال، المسارات، وملفات `.lovable`.
   - يستبعد الملفات الخطرة أو غير المناسبة للتصدير:
     - `.env`
     - `.git`
     - `node_modules`
     - `dist`
     - كاشات البناء
     - أي ملف يحتوي قيماً سرية مباشرة

2. **ملف Secrets Inventory بدون القيم**
   - ملف باسم `SECRETS-INVENTORY.md` يحتوي أسماء الأسرار المطلوبة لإعادة تشغيل المشروع، مثل:
     - `FAL_API_KEY`
     - `LOVABLE_API_KEY`
     - `TELEGRAM_API_KEY`
     - `NOTIFY_WEBHOOK_SECRET`
     - `VIDEO_PROVIDER_CALLBACK_SECRET`
     - `RESEND_API_KEY`
     - `REPLICATE_API_TOKEN`
     - مفاتيح Lovable Cloud الخاصة بالخادم
   - سيوضح وظيفة كل سر وأين يُستخدم، لكن بدون عرض قيمته.

3. **ملف يدوي لحفظ الأسرار خارج المشروع**
   - ملف إرشادي باسم `HOW-TO-SAVE-SECRETS-SAFELY.md` يشرح لك بالضبط أين تحفظ القيم الحقيقية بشكل آمن خارج ZIP، مثل:
     - Bitwarden
     - 1Password
     - iCloud Keychain
     - Google Password Manager
     - ملف مشفر بكلمة مرور على جهازك فقط
   - سأضع لك قالباً تنسخه إلى مدير كلمات المرور، مثل:

```text
RIFD Project Secrets
Admin email: [اكتبها أنت]
Admin password: [اكتبها أنت]
FAL_API_KEY: [اكتبها أنت]
RESEND_API_KEY: [اكتبها أنت]
TELEGRAM_API_KEY: [اكتبها أنت]
REPLICATE_API_TOKEN: [اكتبها أنت]
NOTIFY_WEBHOOK_SECRET: [اكتبها أنت]
VIDEO_PROVIDER_CALLBACK_SECRET: [اكتبها أنت]
```

4. **ملف روابط المشروع والدومينات**
   - `PROJECT-URLS-AND-DOMAINS.md` ويتضمن:
     - رابط المعاينة
     - رابط النشر
     - الدومين المخصص `rifd.site`
     - `www.rifd.site`
     - تنبيه أن `rifd.club` محذوف وممنوع استخدامه.

5. **ملف خطوات الاستعادة**
   - `RESTORE-GUIDE.md` يشرح:
     - كيف ترفع المشروع إلى GitHub كمستودع Private.
     - كيف تعيد تحميله على جهاز آخر.
     - كيف تعيد إدخال الأسرار في Lovable Cloud.
     - كيف تتحقق أن الموقع يعمل بعد الاستعادة.

## النتيجة النهائية

سيتم تسليم ملف ZIP واحد داخل ملفات التنزيل باسم مقترح:

```text
rifd-complete-safe-backup-2026-04-27.zip
```

ويحتوي المشروع كاملاً مع ملفات توثيق تساعدك على إعادة تشغيله، لكن بدون تسريب قيم الأسرار.

## ملاحظة مهمة

إذا أردت الاحتفاظ بالقيم الحقيقية للأسرار، يجب أن تنسخها بنفسك من أماكنها الأصلية أو من مدير الأسرار إلى مدير كلمات مرور آمن. أنا أستطيع توثيق أسمائها ووظائفها وخطوات استعادتها، لكن لا أستطيع استخراجها أو كتابتها داخل ZIP.
