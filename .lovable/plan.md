أوافق على دمج الحزمتين الثالثة والرابعة الآن؛ التكامل بين “بيت الحملة الحي” ونسخ A/B وتقويم 7 أيام سيحوّل الاستوديو فعلاً من صفحة تخطيط إلى مركز قيادة تشغيلي.

**نسبة تقدم خطة استوديو الحملات:** 100% للحزمتين الثالثة والرابعة بعد إضافة بيت الحملة الحي، نسخ A/B، تقويم 7 أيام، مؤشرات الإدارة، وربط المكتبة باكتمال 3/3. يبقى البريد مؤجلاً لمرحلة ما قبل الإطلاق العام ولا يؤثر على هذه النسبة.

لكن كمراجعة مهنية: الخطة قوية جداً، لكنها تحتاج ضبطين مهمين حتى تكون SaaS-grade وليست مجرد واجهة جميلة:

1. لا يكفي عرض نسخ A/B للنسخ فقط؛ يجب أن يستطيع المستخدم استخدام النسخة المختارة مباشرة داخل توليد النص أو كمدخل للحملة.
2. لا يكفي إنشاء تقويم نشر كقائمة؛ يجب أن يكون قابلاً للتنفيذ: نسخ رسالة اليوم، وفتح أداة النص/الصورة/الفيديو من يوم التقويم مع سياق الحملة.

سأنفذ ذلك ضمن نفس الروح، بدون إضافة قياس نتائج أو نظام جدولة فعلي أو نشر تلقائي الآن.

## قرار البريد

سأزيل أي اعتبار للبريد كمانع للحزمتين 3 و4.

- البريد مؤجل قبل الإطلاق العام فقط.
- لا توجد أي وظيفة في الحزمتين 3 و4 تعتمد على إرسال بريد.
- لن أطلب إعداد Cloud → Emails أثناء تنفيذ هذه الحزم.

## المرحلة 0: تدقيق سريع قبل البناء

قبل التعديل سأراجع عملياً نقاط الحزمة الثانية التي سنبني فوقها:

- `campaignId` هو الاسم الأساسي في الواجهة.
- `campaignPackId` يبقى للتوافق فقط.
- فتح حملة محفوظة يعيد تعبئة الاستوديو.
- أدوات النص/الصورة/الفيديو تحفظ metadata للحملة.
- المكتبة تقرأ مفاتيح metadata الثلاثة.
- لا توجد روابط صامتة أو imports مكررة في الملفات المتأثرة.

أي كود مكرر أو قديم سأحذفه وأذكره في ملخص التنفيذ.

## المرحلة 1: توسيع نموذج Campaign Pack

لأن نسخ A/B وتقويم النشر يجب أن يبقيا محفوظين عند الرجوع للحملة، سأضيف حقولاً منظمة إلى `campaign_packs` بدلاً من دفنها كنص داخل `brief`.

### التغيير البنيوي المقترح

إضافة عمودين:

- `ab_variants jsonb not null default '[]'`
- `publishing_calendar jsonb not null default '[]'`

مع تحديث RLS validation لتحديد الحجم ومنع payload كبير.

### السبب

هذا أفضل من تخزينها داخل نص `brief` لأن:

- الواجهة تستطيع عرض البطاقات بدون parsing هش.
- الإدارة تستطيع لاحقاً قراءة عدد النسخ والأيام.
- لا تنكسر الحملات القديمة؛ الافتراضي مصفوفة فارغة.

## المرحلة 2: توسيع `generateCampaignBrief`

في `src/server/campaign-packs.ts` سأوسّع المخرجات المنظمة لتشمل:

### `abVariants`

ثلاث نسخ تسويقية، كل نسخة بالشكل:

```ts
{
  style: "مباشرة" | "عاطفية" | "ندرة/عرض" | string,
  icon: string,
  hook: string,
  message: string,
  cta: string
}
```

مع إلزام النموذج بإنتاج 3 نسخ مختلفة فعلاً:

- نسخة مباشرة للبيع.
- نسخة مبنية على الألم/الرغبة.
- نسخة عرض/ندرة/فرصة.

### `publishingCalendar`

خطة 7 أيام بالشكل:

```ts
{
  day: number,
  label: string,
  channel: string,
  contentType: string,
  message: string,
  goal: string
}
```

مع ضبطها للسوق السعودي:

- لغة واضحة غير مبالغ فيها.
- قنوات مناسبة لاختيار المستخدم.
- أهداف عملية: وعي، تفاعل، إثبات، تحويل، إعادة تذكير.

### التوافق مع الحملات القديمة

الحملات القديمة التي لا تحتوي `ab_variants` أو `publishing_calendar` ستعرض:

- رسالة “أعد بناء الخطة لتوليد نسخ A/B وتقويم نشر”.
- بدون انهيار أو شاشة فارغة.

## المرحلة 3: Server Function لجلب بيت الحملة الحي

سأضيف دالة خادمية آمنة في `src/server/campaign-packs.ts` تقرأ مخرجات الحملة الحالية فقط:

- من `generations`:
  - أول/آخر نص مرتبط بالحملة.
  - أول/آخر صورة مرتبطة بالحملة.
- من `video_jobs`:
  - أحدث فيديو مرتبط بالحملة.
  - حالته: pending / processing / completed / failed / refunded.

ستقرأ الربط من metadata بالمفاتيح:

- `campaignId`
- `campaignPackId`
- `campaign_pack_id`

### الأمان

- المستخدم يرى حملاته ومخرجاته فقط.
- الأدمن في صفحات الإدارة يستخدم دوال admin منفصلة بتحقق خادمي.
- لا اعتماد على localStorage أو صلاحيات من الواجهة.

## المرحلة 4: بيت الحملة الحي داخل Campaign Studio

في `src/routes/dashboard.campaign-studio.tsx` سأستبدل بطاقات الوجهة الساكنة عند وجود `campaignId` بحالة حية.

### فتحة النص

إذا وُجد نص:

- علامة نجاح خضراء.
- أول 100 حرف من النص.
- زر “عرض/تعديل” يفتح أداة النص مع:
  - `campaignId`
  - prompt الحملة
  - ويمكن تمرير النص كمرجع إن كان مناسباً بدون كسر الأداة.

إذا لم يوجد:

- حالة “لم يُنفذ بعد”.
- زر “اكتب نصاً يبيع”.

### فتحة الصورة

إذا وُجدت صورة:

- Thumbnail مربع/أفقي حسب المتاح.
- علامة نجاح خضراء.
- زر “عرض/تعديل” يفتح أداة الصورة بنفس `campaignId`.

إذا لم توجد:

- زر “صمّم صورة إعلان”.

### فتحة الفيديو

إذا وُجد فيديو:

- `pending/processing`: Spinner + “قيد المعالجة”.
- `completed`: زر “مشاهدة”.
- `failed/refunded`: حالة مفهومة مع زر “أنشئ فيديو جديد”.

إذا لم يوجد:

- زر “أنشئ فيديو قصير”.

### نسبة تقدم الحملة

سأضيف شريط تقدم للحملة نفسها:

```text
0/3  = 0%
1/3  = 33%
2/3  = 67%
3/3  = 100%
```

مع تسمية واضحة حتى لا تختلط مع “تقدم الخطة” أو “تقدم الحزمة”.

## المرحلة 5: نسخ A/B في الواجهة

تحت الموجز المنظم داخل `CampaignHouse` سأضيف قسم:

“اختر أقوى رسالة تسويقية”

لكل بطاقة:

- الأيقونة.
- نوع النسخة: مباشرة / عاطفية / ندرة أو غيرها.
- Hook.
- Message.
- CTA.
- زر “نسخ الرسالة”.
- زر عملي إضافي: “استخدمها في النص” يفتح أداة النص مع prompt مركّب من النسخة + سياق الحملة.

هذا يجعل A/B قابلاً للتنفيذ وليس مجرد عرض.

## المرحلة 6: تقويم النشر 7 أيام

سأضيف قسم جديد داخل بيت الحملة:

“تقويم نشر 7 أيام”

كل يوم يظهر كبطاقة صغيرة أو Timeline متجاوب:

- اليوم والاسم.
- القناة.
- نوع المحتوى.
- الرسالة.
- الهدف.
- زر “نسخ رسالة اليوم”.
- زر “حوّلها لمحتوى” يفتح أداة النص مع `campaignId` ورسالة اليوم.

### حدود النطاق

لن أضيف الآن:

- جدولة فعلية.
- اتصال بمنصات اجتماعية.
- reminders.
- قياس نتائج.

هذا يبقى تقويم تنفيذ يدوي ذكي داخل الاستوديو.

## المرحلة 7: تحديث صفحة Admin Campaigns

في `src/routes/admin.campaign-packs.tsx` و`src/server/campaign-packs.ts` سأحسّن إدارة الحملات لتعرض ما يفيد التشغيل:

- فتح الحملة باستخدام `campaignId` بدلاً من مجرد إعادة تعبئة الحقول.
- عدد المخرجات المرتبطة:
  - نصوص
  - صور
  - فيديوهات
- نسبة اكتمال الحملة.
- عدد نسخ A/B.
- عدد أيام التقويم.
- آخر نشاط.

هذا يجعل الإدارة ترى هل المستخدمون فعلاً ينفذون الحملات أم يكتفون بتوليد الخطة.

## المرحلة 8: تحسين Admin Video Jobs للحملات

في صفحة إدارة الفيديو سأبرز سياق الحملة إن وجد:

- اسم/منتج الحملة من metadata.
- Badge “مرتبط بحملة”.
- رابط فتح الحملة في Campaign Studio أو Admin Campaigns.

هذا مهم تشغيلياً لأن الفيديو هو الأصل الأعلى تكلفة.

## المرحلة 9: مراجعة المكتبة بعد التوسيع

**الحالة:** مكتملة — المكتبة تعرض الآن مجموعات الحملات مع نسبة اكتمال 3/3 ورابط فتح بيت الحملة.

سأحافظ على تجميع المكتبة الحالي، مع تحسين بسيط إن لزم:

- الحملة المكتملة تظهر كـ “3/3 مخرجات”.
- لا أكسر الفلاتر الحالية.
- لا أعيد تصميم المكتبة بالكامل.

## المرحلة 10: مراجعة النصوص تسويقياً

سأراجع كل النصوص الجديدة لتتبع لغة المنتج:

- “اكتب نصاً يبيع”
- “صمّم صورة إعلان”
- “أنشئ فيديو قصير”
- “اختر أقوى رسالة تسويقية”
- “تقويم نشر 7 أيام”
- “حوّلها لمحتوى”

وسأتجنب:

- وعود رقمية غير مثبتة.
- عبارات عامة مثل “أفضل حملة على الإطلاق” داخل المنتج.
- نصوص لا تناسب السوق السعودي.
- هلوسة مثل نسب تحويل أو نتائج مضمونة.

## المرحلة 11: التجاوب وDark/Light Mode

سأبني المكونات الجديدة Mobile-first:

- شبكة 3 أعمدة على Desktop فقط.
- تقويم 7 أيام كسكرول/شبكة بدون overflow أفقي.
- ألوان من theme tokens فقط.
- دعم RTL كامل.
- أزرار مناسبة للمس.

## المرحلة 12: التوثيق ونسبة التقدم

سأحدّث التوثيق الداخلي بحيث يوضح:

- الحزمة الثانية مكتملة.
- الحزمة الثالثة والرابعة قيد التنفيذ/مكتملتان بعد التطبيق.
- البريد مؤجل وليس مانعاً للبناء.
- نسبة تقدم خطة الاستوديو بعد الحزمتين.

في الواجهة سأصحح أي تسمية قديمة مثل “تقدّم الحزمة الثانية” إن أصبحت مضللة، إلى تسمية أشمل مثل:

- “تقدّم مركز الحملات”
- أو “جاهزية الاستوديو”

## نقد الخطة: هل تصل بنا إلى أفضل استوديو حملات؟

نعم، هذه الحزمتان سترفعان المنتج نقلة كبيرة لأنهما تضيفان ثلاث طبقات مهمة:

1. طبقة التنفيذ: حالة النص/الصورة/الفيديو داخل بيت الحملة.
2. طبقة الذكاء التسويقي: نسخ A/B قابلة للاستخدام.
3. طبقة التشغيل: تقويم 7 أيام يحوّل الخطة إلى خطوات يومية.

لكن للوصول إلى “أفضل استوديو حملات” فعلاً ستبقى حزم مستقبلية لازمة، ولا أنفذها الآن:

- قياس أداء النسخ A/B بعد النشر.
- جدولة تلقائية أو تذكير بالنشر.
- حفظ النسخة المختارة كـ “winning angle”.
- ربط الحملة بالمبيعات أو UTM لاحقاً.
- تقارير ROI بسيطة للتاجر.

الحزمتان الثالثة والرابعة هما الأساس الصحيح لهذه المراحل، ولن أضيف تلك المراحل الآن إلا إذا طلبتها لاحقاً.

## معايير القبول بعد التنفيذ

- فتح `/dashboard/campaign-studio?campaignId=X` يعرض بيت حملة حي وليس بطاقات ساكنة فقط.
- النص يظهر بمقتطف 100 حرف عند وجوده.
- الصورة تظهر thumbnail عند وجودها.
- الفيديو يظهر حالة معالجة أو اكتمال أو فشل بوضوح.
- نسخ A/B الثلاث تظهر وتُنسخ وتُستخدم في أداة النص.
- تقويم 7 أيام يظهر ويمكن نسخ/تحويل رسالة كل يوم لمحتوى.
- Admin Campaigns يعرض مؤشرات تنفيذ الحملة.
- Admin Video Jobs يبرز ربط الفيديو بالحملة.
- لا يوجد اعتماد على البريد.
- لا يوجد كسر على Mobile/Tablet/Desktop أو Dark/Light Mode.
